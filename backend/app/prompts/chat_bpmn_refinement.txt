## Instrucciones
1. Analiza si el mensaje implica cambios al diagrama BPMN.
2. Si hay cambios, devuelve el modelo BPMN completo actualizado (estructura process con pools, lanes, elements, flows).
3. Mantén IDs estables cuando sea posible; usa IDs nuevos solo para elementos nuevos.
4. Valida: exactamente 1 startEvent, al menos 1 endEvent, todos los flows conectados.
5. Responde en español en el campo "reply".

Responde SOLO con JSON:

```json
{
  "model_updated": true,
  "reply": "Explicación de los cambios realizados al diagrama...",
  "changes_summary": ["Cambio 1", "Cambio 2"],
  "bpmn_model": {
    "process": {
      "id": "proc_001",
      "name": "Nombre del Proceso",
      "type": "macro",
      "pools": [],
      "elements": [],
      "flows": []
    }
  },
  "pending_questions": ["Pregunta opcional si falta información"]
}
```

Si el mensaje NO requiere cambios al diagrama (solo consulta), usa model_updated: false y omite bpmn_model.
